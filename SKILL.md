# Travel Itinerary Generator

## Purpose

帮助用户生成完整的、可视化的HTML旅行行程单。

**支持两种使用场景：**

### 场景A：从零规划（还没订机票酒店）
用户只需要说：
- 想去哪里
- 什么时间
- 几个人
- 喜欢什么类型的景点

AI 会：
- 推荐住宿区域和酒店选择建议
- 根据偏好安排景点和路线
- 生成完整行程单（酒店标记为"待预订"）

### 场景B：整理已有信息（已订机票酒店）
用户提供：
- 机票截图/信息
- 酒店截图/信息
- 想去的景点

AI 会：
- 提取关键信息
- 整合成完整行程单
- 标记预订状态

**核心价值：**
- 一站式信息整合：机票、酒店、景点、预算、签证、天气、餐饮全覆盖
- 可视化呈现：SVG地图路线、时间轴日程、卡片式布局
- 实用导向：每个模块都服务于"出行前准备"和"旅途中查阅"

---

## When to Use

- 用户想去某个地方旅行，还在规划阶段
- 用户已订好机票酒店，需要整理成行程单
- 用户需要一份可打印/可分享的完整行程单
- 用户说"帮我做一个旅行行程单"、"我想去XX玩"

## 场景A：偏好探索交互流程

当用户处于"从零规划"阶段时，通过一系列问答了解用户偏好，然后推荐合适的景点和行程节奏。

### 交互流程设计

**第一轮：基础信息**
```
Q1: 想去哪里？（国家/城市）
Q2: 什么时间？几天？
Q3: 几个人？
```

**第二轮：旅行风格探索**
```
Q4: 你理想的旅行节奏是？
    - 悠闲慢游（每天2-3个点，留足吃喝发呆时间）
    - 适中平衡（每天3-4个点，有节奏但不累）
    - 紧凑充实（每天5+个点，不想错过任何东西）

Q5: 这次旅行最想体验什么？（可多选）
    - 🏛️ 人文历史（博物馆、教堂、古迹）
    - 📸 网红打卡（出片、地标、热门景点）
    - 🍽️ 美食探店（当地特色、米其林、街头小吃）
    - 🛍️ 购物血拼（商场、买手店、当地品牌）
    - 🌿 自然风光（公园、海边、徒步）
    - 🎭 艺术文化（画廊、演出、音乐）
    - 🍺 夜生活（酒吧、夜景、夜市）
    - 🧘 休闲放松（咖啡馆、温泉、慢生活）
```

**第三轮：体力与限制**
```
Q6: 你们的体力情况？
    - 能走！一天2万步不在话下
    - 一般般，走太多会累
    - 希望少走路，多坐车或打车

Q7: 有没有必去的地方或必做的事？
    （开放式输入，如："一定要去天文钟""想喝正宗捷克啤酒"）

Q8: 有没有不想要的？
    （如："不想去博物馆""不想排长队"）
```

**第四轮：预算确认**
```
Q9: 预算大概是什么水平？
    - 💰 穷游省钱（能省则省，体验为主）
    - 💰💰 中等舒适（该花花，不奢侈）
    - 💰💰💰 品质优先（住好吃好，体验升级）
```

### 偏好 → 景点推荐匹配逻辑

根据用户回答，AI 进行以下匹配：

**旅行节奏 → 每日景点数量**
| 节奏偏好 | 每日安排 | 单个景点时长 |
|---------|---------|-------------|
| 悠闲慢游 | 2-3个点 | 充裕，含休息 |
| 适中平衡 | 3-4个点 | 标准时长 |
| 紧凑充实 | 5-6个点 | 精简高效 |

**体验偏好 → 景点类型筛选**
| 用户选择 | 推荐景点类型 | 不推荐 |
|---------|-------------|--------|
| 人文历史 | 博物馆、教堂、城堡、老城区 | 购物中心 |
| 网红打卡 | 地标建筑、观景台、网红咖啡店 | 冷门小众点 |
| 美食探店 | 本地餐厅、市场、美食街 | 赶场式景点 |
| 购物血拼 | 商业区、奥特莱斯、特色小店 | 偏远景点 |
| 自然风光 | 公园、海滩、郊区、徒步路线 | 室内景点 |
| 夜生活 | 酒吧街、夜景观景点、夜市 | 早起行程 |
| 休闲放松 | 咖啡馆、温泉、慢节奏街区 | 热门排队点 |

**体力情况 → 行程密度调整**
| 体力 | 调整策略 |
|-----|---------|
| 能走 | 可安排徒步、爬山、步行串联景点 |
| 一般 | 景点之间安排休息，避免连续爬坡 |
| 少走 | 优先推荐有公共交通/可打车的景点，减少步行 |

**必去/不想要 → 硬性约束**
- 必去的地方：优先安排在最佳时段（如日落观景点安排在傍晚）
- 不想要的：从候选景点中剔除

### 输出：个性化推荐

完成问答后，AI 输出：

```
🎯 根据你的偏好，我为你定制了这趟旅行：

【旅行风格】悠闲慢游 + 人文历史 + 美食探店

✅ 查理大桥日落（必去·拍照绝佳）
✅ 天文钟（你指定的必去）
✅ 本地啤酒馆 U Fleků（美食体验）
✅ Café Louvre 百年咖啡馆（休闲放松）

【不安排的景点】

确认这个方向吗？我来生成完整行程单。
```

## Hotel Recommendation Logic

当用户处于规划阶段，AI 根据以下因素推荐住宿区域：

### 1. 景点偏好 → 住宿区域匹配

| 偏好类型 | 推荐住宿区域逻辑 |
|---------|-----------------|
| 自然风光 | 靠近自然景点的交通枢纽 |
| 人文历史 | 老城区/历史街区附近 |
| 网红打卡 | 核心景点集中区 |
| 美食购物 | 商业中心/美食街附近 |
| 亲子游 | 交通便利+安静的区域 |
| 小众深度 | 本地人生活区 |

### 2. 行程动线 → 最优位置

- 分析用户想去的景点分布
- 推荐能辐射最多景点的中心位置
- 考虑公共交通便利性

### 3. 预算 → 住宿类型

| 预算 | 推荐类型 |
|-----|---------|
| 穷游 | 青旅/胶囊酒店/郊区民宿 |
| 中等 | 民宿/经济型酒店 |
| 舒适 | 品牌酒店/精品民宿 |

## Workflow

### Step 1: 收集信息
向用户询问必要输入信息，使用结构化问题引导：
- "请告诉我这次旅行的基本信息：目的地国家、城市顺序、日期范围、出行人数"
- "请提供每个城市的酒店预订信息"
- "请提供机票信息"

### Step 2: 确认信息
整理用户提供的信息，列出清单让用户确认：
- 城市顺序和日期
- 酒店信息
- 机票信息
- 景点列表

### Step 3: 生成HTML
按照Output Structure生成完整HTML文件，保存到用户指定路径。

### Step 4: 二次检查（自动执行）

生成行程单后，AI **必须自动执行以下检查**，并向用户报告问题和优化建议：

#### 4.1 季节/天气适配检查

| 检查项 | 规则 |
|-------|------|
| 夏季高温（>30°C） | 上午10点-下午4点避免户外暴晒景点，优先安排室内（商场/博物馆/水族馆） |
| 冬季严寒（<0°C） | 户外景点控制在1-2小时内，穿插室内暖场 |
| 雨季/台风季 | 备注雨天备选方案，避免全天户外 |
| 日落时间 | 观景/海边/日落景点安排在实际日落时段（需查当地日落时间） |

**输出格式**：
```
🌡️ 季节适配检查：
- ✅ Day 2 中午安排室内商场避暑，合理
- ⚠️ Day 3 下午2点安排甘川文化村（户外），建议改到傍晚4点后
```

#### 4.2 路线顺路检查

| 检查项 | 规则 |
|-------|------|
| 地理位置 | 同一天景点应在同一区域或沿线分布，避免来回折返 |
| 最优顺序 | 检查是否有更顺的排列方式（如顺时针/逆时针绕圈） |
| 酒店位置 | 每天最后一站应方便回酒店（有地铁/好打车） |

**输出格式**：
```
🗺️ 路线顺路检查：
- ✅ Day 2 海云台→青沙浦→广安里，东部沿线顺路
- ⚠️ Day 3 原路线松岛→南浦洞→甘川→白浅滩有绕路
  → 建议优化为：南浦洞→松岛→甘川→白浅滩（顺时针）
```

#### 4.3 交通方式推荐

| 场景 | 推荐交通 |
|-----|---------|
| 2公里内 | 步行 |
| 地铁沿线 | 地铁（标注线路和换乘） |
| 非地铁覆盖 | 公交或打车（标注预估费用和时间） |
| 跨城/郊区 | 火车/长途巴士/包车 |

**输出格式**：
```
🚌 交通方式推荐：
- 西面→海云台：地铁2号线，约40分钟，¥14
- 松岛→甘川：打车，约15分钟，¥30
- 白浅滩→南浦洞：公交26路，约20分钟，¥8
```

#### 4.4 每日收尾检查

| 检查项 | 规则 |
|-------|------|
| 最后一站 | 必须方便回酒店（有地铁站或好打车） |
| 晚餐时间 | 避免安排在偏僻景点后（可能找不到餐厅） |
| 营业时间 | 检查最后一站营业时间是否覆盖计划时段 |

**输出格式**：
```
🏠 每日收尾检查：
- ✅ Day 2 广安里有地铁2号线，回西面方便
- ⚠️ Day 3 白浅滩无地铁，建议结束后打车去南浦洞吃晚餐再地铁回
```

#### 4.5 检查结果汇总

完成所有检查后，输出汇总：
```
📋 二次检查汇总：
- 季节适配：2处建议优化
- 路线顺路：1处绕路已优化
- 交通方式：已标注所有两点间最佳交通
- 每日收尾：1处建议调整
- 潜在踩坑：3处风险提醒

是否按建议优化行程？
```

用户确认后，自动更新行程单。

#### 4.6 潜在踩坑预警

主动发现行程中可能遇到的坑，提前告知用户并给出规避建议。

**常见踩坑类型**：

| 类型 | 检查规则 | 示例 |
|-----|---------|------|
| 营业时间 | 检查景点/餐厅营业时间是否覆盖计划时段 | "白浅滩餐厅19:30后不接单" |
| 预约/限流 | 检查是否需要提前预约或有限流 | "蓝线胶囊火车必须提前订票，热门时段秒光" |
| 节假日闭馆 | 检查是否遇到当地节假日/休息日 | "周一博物馆闭馆" |
| 排队时间 | 热门景点排队预估 | "天文钟整点报时前30分钟开始排队" |
| 交通陷阱 | 去机场/火车站的交通坑 | "去机场打车要¥110，地铁只要¥25" |
| 支付方式 | 是否只收现金/不支持外卡 | "市场摊位只收现金，刷不了卡" |
| 天气依赖 | 景点体验强依赖天气 | "峡湾游船下雨体验很差，建议备选室内行程" |
| 时间陷阱 | 容易低估的时间 | "从甘川到白浅滩看着近，但山路绕，实际要20分钟" |
| 季节限定 | 部分项目/景点季节性开放 | "松岛缆车冬季检修停运" |
| 安全提醒 | 治安、诈骗、常见套路 | "出租车绕路常见，建议用打车App" |

**输出格式**：
```
⚠️ 潜在踩坑预警：

1. 🎫 蓝线火车必须提前订票
   - 风险：热门时段当天买不到票
   - 建议：提前3天在官网/Klook预订16:00场次

2. 🍽️ 白浅滩餐厅19:30后不接单
   - 风险：看完日落找不到晚餐
   - 建议：在白浅滩吃早点，或去南浦洞吃晚餐

3. 💰 去机场打车很贵（¥110+）
   - 风险：现金不够或被宰
   - 建议：地铁2号线到沙上站换轻轨，3人只要¥25

4. 🏪 市场/路边摊只收现金
   - 风险：没现金吃不了街边小吃
   - 建议：提前换好现金，人均6万韩元够用
```

**检查来源**：
- 目的地常见踩坑攻略（小红书、马蜂窝、穷游）
- 当地节假日/营业时间数据
- 交通费用对比
- 季节性限制信息

### Step 5: 迭代优化
根据用户反馈调整：
- 更新价格
- 添加/删除景点
- 调整日程时间
- 完善位置介绍
