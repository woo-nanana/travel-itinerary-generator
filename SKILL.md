# Travel Itinerary Generator

## Purpose

帮助用户生成完整的、可视化的HTML旅行行程单。

**支持两种使用场景：**

### 场景A：从零规划（还没订机票酒店）
用户只需要说：
- 想去哪里
- 什么时间
- 几个人
- 喜欢什么类型的景点

AI 会：
- 推荐住宿区域和酒店选择建议
- 根据偏好安排景点和路线
- 生成完整行程单（酒店标记为"待预订"）

### 场景B：整理已有信息（已订机票酒店）
用户提供：
- 机票截图/信息
- 酒店截图/信息
- 想去的景点

AI 会：
- 提取关键信息
- 整合成完整行程单
- 标记预订状态

**核心价值：**
- 一站式信息整合：机票、酒店、景点、预算、签证、天气、餐饮全覆盖
- 可视化呈现：SVG地图路线、时间轴日程、卡片式布局
- 实用导向：每个模块都服务于"出行前准备"和"旅途中查阅"

---

## When to Use

- 用户想去某个地方旅行，还在规划阶段
- 用户已订好机票酒店，需要整理成行程单
- 用户需要一份可打印/可分享的完整行程单
- 用户说"帮我做一个旅行行程单"、"我想去XX玩"

## 场景A：偏好探索交互流程

当用户处于"从零规划"阶段时，通过一系列问答了解用户偏好，然后推荐合适的景点和行程节奏。

### 交互流程设计

**第一轮：基础信息**
```
Q1: 想去哪里？（国家/城市）
Q2: 什么时间？几天？
Q3: 几个人？
```

**第二轮：旅行风格探索**
```
Q4: 你理想的旅行节奏是？
    - 悠闲慢游（每天2-3个点，留足吃喝发呆时间）
    - 适中平衡（每天3-4个点，有节奏但不累）
    - 紧凑充实（每天5+个点，不想错过任何东西）

Q5: 这次旅行最想体验什么？（可多选）
    - 🏛️ 人文历史（博物馆、教堂、古迹）
    - 📸 网红打卡（出片、地标、热门景点）
    - 🍽️ 美食探店（当地特色、米其林、街头小吃）
    - 🛍️ 购物血拼（商场、买手店、当地品牌）
    - 🌿 自然风光（公园、海边、徒步）
    - 🎭 艺术文化（画廊、演出、音乐）
    - 🍺 夜生活（酒吧、夜景、夜市）
    - 🧘 休闲放松（咖啡馆、温泉、慢生活）
```

**第三轮：体力与限制**
```
Q6: 你们的体力情况？
    - 能走！一天2万步不在话下
    - 一般般，走太多会累
    - 希望少走路，多坐车或打车

Q7: 有没有必去的地方或必做的事？
    （开放式输入，如："一定要去天文钟""想喝正宗捷克啤酒"）

Q8: 有没有不想要的？
    （如："不想去博物馆""不想排长队"）
```

**第四轮：预算确认**
```
Q9: 预算大概是什么水平？
    - 💰 穷游省钱（能省则省，体验为主）
    - 💰💰 中等舒适（该花花，不奢侈）
    - 💰💰💰 品质优先（住好吃好，体验升级）
```

### 偏好 → 景点推荐匹配逻辑

根据用户回答，AI 进行以下匹配：

**旅行节奏 → 每日景点数量**
| 节奏偏好 | 每日安排 | 单个景点时长 |
|---------|---------|-------------|
| 悠闲慢游 | 2-3个点 | 充裕，含休息 |
| 适中平衡 | 3-4个点 | 标准时长 |
| 紧凑充实 | 5-6个点 | 精简高效 |

**体验偏好 → 景点类型筛选**
| 用户选择 | 推荐景点类型 | 不推荐 |
|---------|-------------|--------|
| 人文历史 | 博物馆、教堂、城堡、老城区 | 购物中心 |
| 网红打卡 | 地标建筑、观景台、网红咖啡店 | 冷门小众点 |
| 美食探店 | 本地餐厅、市场、美食街 | 赶场式景点 |
| 购物血拼 | 商业区、奥特莱斯、特色小店 | 偏远景点 |
| 自然风光 | 公园、海滩、郊区、徒步路线 | 室内景点 |
| 夜生活 | 酒吧街、夜景观景点、夜市 | 早起行程 |
| 休闲放松 | 咖啡馆、温泉、慢节奏街区 | 热门排队点 |

**体力情况 → 行程密度调整**
| 体力 | 调整策略 |
|-----|---------|
| 能走 | 可安排徒步、爬山、步行串联景点 |
| 一般 | 景点之间安排休息，避免连续爬坡 |
| 少走 | 优先推荐有公共交通/可打车的景点，减少步行 |

**必去/不想要 → 硬性约束**
- 必去的地方：优先安排在最佳时段（如日落观景点安排在傍晚）
- 不想要的：从候选景点中剔除

### 输出：个性化推荐

完成问答后，AI 输出：

```
🎯 根据你的偏好，我为你定制了这趟旅行：

【旅行风格】悠闲慢游 + 人文历史 + 美食探店
【每日节奏】2-3个景点，留足吃喝时间
【住宿建议】老城区（步行可达核心景点）

【为你推荐的景点】
✅ 布拉格城堡（必去·人文历史）
✅ 查理大桥日落（必去·拍照绝佳）
✅ 天文钟（你指定的必去）
✅ 本地啤酒馆 U Fleků（美食体验）
✅ Café Louvre 百年咖啡馆（休闲放松）
...

【不安排的景点】
❌ 犹太区博物馆（你说不想去博物馆）
❌ CK小镇一日游（车程较长，与悠闲风格冲突）

确认这个方向吗？我来生成完整行程单。
```

## Hotel Recommendation Logic

当用户处于规划阶段，AI 根据以下因素推荐住宿区域：

### 1. 景点偏好 → 住宿区域匹配

| 偏好类型 | 推荐住宿区域逻辑 |
|---------|-----------------|
| 自然风光 | 靠近自然景点的交通枢纽 |
| 人文历史 | 老城区/历史街区附近 |
| 网红打卡 | 核心景点集中区 |
| 美食购物 | 商业中心/美食街附近 |
| 亲子游 | 交通便利+安静的区域 |
| 小众深度 | 本地人生活区 |

### 2. 行程动线 → 最优位置

- 分析用户想去的景点分布
- 推荐能辐射最多景点的中心位置
- 考虑公共交通便利性

### 3. 预算 → 住宿类型

| 预算 | 推荐类型 |
|-----|---------|
| 穷游 | 青旅/胶囊酒店/郊区民宿 |
| 中等 | 民宿/经济型酒店 |
| 舒适 | 品牌酒店/精品民宿 |

## Output Structure

生成的HTML行程单包含以下模块（按顺序）：

### 1. Header 头部
- 旅行标题
- 副标题（国家/主题）
- 日期徽章

### 2. SVG Map 路线地图
- 途经国家/城市轮廓
- 城市/区域标记点
- 交通路线
- 图例

### 3. Table of Contents 目录
- 快速跳转锚点

### 4. City Overview 城市景点总览
- 每个城市/每天一张卡片
- 列出景点安排

### 5. Main Schedule 景点/活动总表
- 按时间顺序排列
- 列：日期、时间、活动、时长、费用
- 高亮规则：每天第一个活动浅绿，航班浅橙

### 6. Flight Info 机票信息
- 所有航段详情表
- 机票总价

### 7. Hotel Details 酒店预订详情
- 每个城市一张卡片
- 已订：酒店名称、地址、价格、预订状态
- 待订：推荐区域、选择建议、参考价格

### 8. Special Activities 特殊活动（如有）

### 9. Budget 预算估算
- 分项卡片
- 总预算
- 现金建议

### 10. Visa Checklist 签证准备清单

### 11. Weather & Clothing 天气及穿搭

### 12. Food Guide 餐饮与超市指南

### 13. Transport Tips 交通支付攻略

### 14. Booking Checklist 预订清单
- 状态标记（已预订/待预订）

### 15. Footer 页脚

## Styling Guidelines

### 配色方案
```css
--primary-blue: #5B8BA0;    /* 主色调 */
--dark: #2C3E50;            /* 文字 */
--light-bg: #F5F7FA;        /* 背景 */
--accent: #E8B86D;          /* 强调色 */
--green: #27AE60;           /* 已完成/已预订 */
--red: #E74C3C;             /* 警告/重要 */
--orange: #F39C12;          /* 提示 */
```

### 国家标识色
- 用国旗emoji区分国家
- 表格中用浅色背景标签

### 高亮规则
- 每天第一个活动：`background: #E8F5E9`（浅绿）
- 航班行程：`background: #FFF5E5`（浅橙）
- 已预订状态：绿色文字 + ✅
- 待预订/重要提示：红色/橙色文字

### 响应式
- 最大宽度1200px居中
- 移动端表格字号缩小
- 卡片自适应网格

## Workflow

### Step 1: 收集信息
向用户询问必要输入信息，使用结构化问题引导：
- "请告诉我这次旅行的基本信息：目的地国家、城市顺序、日期范围、出行人数"
- "请提供每个城市的酒店预订信息"
- "请提供机票信息"

### Step 2: 确认信息
整理用户提供的信息，列出清单让用户确认：
- 城市顺序和日期
- 酒店信息
- 机票信息
- 景点列表

### Step 3: 生成HTML
按照Output Structure生成完整HTML文件，保存到用户指定路径。

### Step 4: 迭代优化
根据用户反馈调整：
- 更新价格
- 添加/删除景点
- 调整日程时间
- 完善位置介绍

## Notes

1. **地图生成**：SVG地图需要根据实际国家/城市位置调整坐标，可参考Google Maps确定相对位置
2. **天气数据**：建议用户提供或查询目的地历史天气数据
3. **签证清单**：根据目的地国家调整（申根/美签/日签等签证要求不同）
4. **货币换算**：统一使用人民币标价，必要时标注当地货币
5. **文件命名**：建议格式 `{目的地}-trip-{年份}.html`
